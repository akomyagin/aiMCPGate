# POST_MVP_PLAN — aiMCPGate

За пределами MVP (Фазы 1 и 2). Не расписано детально — это очередь идей на
будущее, приоритизируется по мере реального спроса. MVP-границы — [`PLAN.md`](PLAN.md) §5.

## Надёжность upstream

- **Health-checks + авто-рестарт** упавших stdio-upstream (экспоненциальный
  backoff, лимит попыток). В MVP упавший upstream просто изолируется.
- **Реагирование на `notifications/tools/list_changed`** — переагрегация
  каталога на лету, когда upstream меняет набор инструментов.
  `[TODO уточнить по официальной спецификации]`
- **Горячая перезагрузка конфига** — добавить/убрать upstream без рестарта шлюза.

## Производительность / масштаб

- **Multi-client** поверх одного шлюза (несколько клиентов одновременно) — в
  Фазе 1 предполагается один клиент.
- **Кэширование** результатов идемпотентных вызовов / каталогов.
- **Пул соединений** к HTTP-upstream.

## Наблюдаемость

- **Метрики** (Prometheus): количество вызовов по upstream/инструменту, latency,
  доля ошибок.
- **Трейсинг** (OpenTelemetry-спаны) вызова сквозь шлюз.
- Ротация файла журнала вызовов.

## Безопасность / доступ

- **Полноценная политика доступа**: allow/deny инструментов и ресурсов по
  клиенту/токену, а не только видимость (базовая версия — в Фазе 2).
- **Аутентификация клиента** к HTTP-шлюзу (токен/mTLS).
- **Opt-in логирование payload** вызовов с явным маскированием секретов — по
  умолчанию payload не пишется (см. TECHNICAL_PLAN §6).
- **Трансформация/фильтрация схем**: скрывать/переименовывать/ограничивать
  отдельные инструменты upstream перед показом клиенту.

## Развёртывание

- **VPS / постоянный HTTP-режим** как поддерживаемый сценарий (образ, systemd/
  compose, инструкция). В MVP цель — $0/мес локально (TECHNICAL_PLAN §7).
- Готовый образ с набором популярных upstream MCP-серверов внутри.

## UX

- Веб-дашборд журнала вызовов (богаче, чем минимальный вью Фазы 2).
- `aimcpgate doctor` — проверка, что все upstream стартуют и отвечают на
  `initialize`.
- Автогенерация сниппета конфига для Claude Code / Cursor из текущего набора
  upstream.
